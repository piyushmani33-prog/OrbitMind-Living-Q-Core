# Animated Trajectory Replay

## Purpose

The animated trajectory replay is a server-rendered Mission Workbench surface for inspecting a
predicted Earth-orbit ground track from deterministic offline replay samples. It is designed to
make the useful result visible first while keeping scientific method and evidence available behind
a collapsed disclosure.

The replay is not live tracking, does not show a true current state, and is not certified for
command, collision, maneuver, safety, approval, or operational decisions.

## Route

`POST /workbench/replay` accepts the same bounded Mission Workbench form as
`POST /workbench/run`.

Supported offline source modes remain:

- allowlisted bundled offline catalog samples; and
- request-local custom TLE text validated through the existing offline TLE path.

Exactly one source mode must be active. Custom TLE values are posted in the request body only; raw
TLE lines are not placed in URLs, rendered in HTML, embedded in replay JSON, persisted, or written
to artifacts.

There is no JSON API route for replay in this slice.

## Server-authoritative samples

The browser displays only `TrajectoryReplayResult` samples generated by `TrajectoryReplayService`.
The service remains the scientific authority for:

- SGP4 propagation from pinned orbital elements;
- frame and Earth-rotation handling;
- WGS84 geodetic latitude, longitude, and altitude;
- observer azimuth, elevation, and slant range; and
- dateline-safe track segmentation.

The same-origin replay controller is served only from `/assets/trajectory-replay.js`, an explicit
allowlisted packaged asset route. JavaScript does not propagate orbits, calculate time systems,
transform coordinates, normalize longitude, interpolate scientific values, or infer visibility. It
only selects an existing sample, moves the marker to supplied SVG coordinates, and updates readouts
from that same sample.

## Sampling interval policy

The server selects one deterministic interval before constructing `TrajectoryReplayRequest`:

- duration up to 6 hours: 15-second samples;
- duration over 6 and up to 12 hours: 30-second samples; and
- duration over 12 and up to 24 hours: 60-second samples.

The selected interval is visible in the page, included in deterministic replay references, and not
changed after request construction. The maximum 24-hour request remains within the replay sample
cap and targets an HTML response comfortably below 1 MiB.

## Presentation projection

The page uses a schematic equirectangular SVG view:

- canonical longitude maps to horizontal SVG position;
- geodetic latitude maps to vertical SVG position;
- latitude and longitude graticules, equator, prime meridian, and dateline guides are local SVG
  elements; and
- each `TrajectoryTrackSegment` is rendered as a separate polyline.

No external map tiles, geographical APIs, fonts, CDNs, or world-outline datasets are used. The
background is a schematic visualization, not an authoritative cartographic map.

Dateline-safe segments from the server are authoritative. The browser does not recompute segment
boundaries and does not draw a false line across the world map from approximately +180 degrees to
-180 degrees.

## Controls

Playback starts paused. Controls include:

- Play/Pause;
- previous and next sample;
- timeline scrubber;
- visualization-speed selector with 0.5x, 1x, 2x, and 4x; and
- sample index/count and UTC timestamp readout.

Playback speed changes only the visual duration. It does not alter scientific timestamps, sample
values, request interval, source identity, or deterministic references.

The marker snaps to actual server-provided samples. There is no browser-side interpolation of
latitude, longitude, altitude, azimuth, elevation, range, or timestamp.

## Scientific readouts

The visible readout always includes:

- UTC timestamp;
- geodetic latitude in degrees;
- canonical longitude in degrees;
- WGS84 altitude in kilometres;
- sample sequence and count; and
- observer azimuth, elevation, and slant range when present.

Source and interval summary includes object identity, safe source identity, source epoch, source
age/offset at replay start, replay start/end, selected sampling interval, sample count, segment
count, observer coordinates, and model identifiers.

## Embedded JSON and script isolation

The page embeds one compact display-only JSON payload in an inert `<template>` element. The payload
is versioned and includes only display data needed by the controller:

- ordered sample sequence numbers;
- UTC timestamps;
- geodetic latitude and canonical longitude;
- WGS84 altitude;
- optional observer azimuth, elevation, and range;
- supplied SVG coordinates;
- server-provided segment memberships;
- observer display coordinates;
- safe source identity; and
- deterministic replay references and limitations.

It does not include raw TLE lines, raw provider content, filesystem paths, environment values,
internal exceptions, database objects, SQLAlchemy models, secrets, or unused Cartesian state.

The JSON is serialized compactly and escaped for HTML/script-block safety, including `<`, `>`, `&`,
U+2028, and U+2029. A value containing `</script>` cannot terminate the data container. The
controller uses `JSON.parse` over `textContent`; it does not use `eval`, `new Function`,
`document.write`, or dynamic `innerHTML`.

Executable replay JavaScript is not inline. Browser pages receive a restrictive Content Security
Policy with `script-src 'self'`, `connect-src 'none'`, `object-src 'none'`, `base-uri 'none'`,
`frame-ancestors 'none'`, and `form-action 'self'`. Existing server-rendered inline CSS remains
allowed while the project still uses page-local styles.

The replay controller route:

- returns only the exact packaged asset;
- has no user-controlled path parameter;
- exposes no directory listing or filesystem path;
- returns `application/javascript; charset=utf-8`;
- sets `X-Content-Type-Options: nosniff`; and
- fails with a fixed response if the packaged asset is unavailable.

## Accessibility

The replay page uses semantic headings, an SVG title and description, associated labels for
controls, keyboard-accessible buttons and form controls, visible focus states, `aria-pressed` on
Play/Pause, and timeline accessible value text.

The replay does not autoplay. It respects reduced-motion preference by staying paused. The static
ground track, current sample, source summary, limitations, and method/evidence disclosure remain
available when JavaScript is unavailable; a `<noscript>` note explains that playback controls
require local JavaScript.

## Failure behavior

Validation and calculation failures return a safe server-rendered error page. The response does not
embed partial replay samples, raw TLE lines, local paths, internal exception details, provider
configuration, or stack traces. The route never silently falls back to ISS when a custom source
fails.

If the embedded payload is malformed in the browser, static content remains visible, playback is
disabled, and a fixed local error is shown. No network recovery is attempted.

## State boundary

The replay route does not persist calculations, create missions, write artifacts, modify source
cache rows, create research memory, or add migrations. It performs no network access and adds no
provider behavior, scheduler, background job, agent, LLM call, quantum behavior, collision analysis,
maneuver recommendation, deployment code, or authentication.

## Future work

Guarded fresh-source integration remains future work and must preserve explicit source enablement,
policy, caching, attribution, freshness labels, and non-live safety language.

Richer map rendering, external map data, live source search, city search, browser geolocation, 3D
globes, video/GIF export, persisted replay products, and public APIs require separate review.
