# Risk Register — OrbitMind Living Q-Core

Severity: Low / Medium / High. Status: Open / Mitigated / Accepted.

| ID | Risk | Sev | Status | Mitigation |
|----|------|-----|--------|------------|
| R-001 | Reference documents ("OrbitMind Living Q-Core", "Feasibility Brief") were absent during Phases 0–2. | Med | **CLOSED 2026-06-19** | **Both documents were delivered, inspected, and hashed on 2026-06-19** (SHA-256 in `REFERENCE_MANIFEST.md`); reconciliation now reflects the actual content (`REFERENCE_RECONCILIATION.md`). No accepted decision was contradicted; one justified change (CelesTrak 2-hour floor) was applied. Resolved date: 2026-06-19. |
| R-012a | CelesTrak GP endpoint/format/cadence not independently verified. | Med | **CLOSED (mitigated) 2026-06-19** | Verified against official GP-data docs as cited by the references (`CELESTRAK_VERIFICATION.md`): hostname, JSON/OMM format, fields, query pattern, 2-hour cadence. Min polling floored at 7200s. Endpoint configurable; network disabled by default. |
| R-002 | Local interpreter is Python 3.14.4, ahead of the 3.12 production baseline; future deps may lack 3.14 wheels. | Med | Mitigated | Phase 0/1 deps verified to resolve as cp314 wheels; production baseline pinned to 3.12 in ADR-0002; `requires-python>=3.12`. |
| R-003 | Sample TLE elements are stale and not live truth; risk of being misread as live satellite data. | High | Mitigated | `test_only` flag + provenance + epistemic labeling; README & API docs state explicitly; SR-05 enforced. |
| R-004 | SGP4 accuracy degrades for epochs far from the TLE epoch; results could mislead. | Med | Mitigated | Verification checks altitude/coordinate sanity; epistemic status = deterministic-calculation (model of reality, not truth); duration bounded. |
| R-005 | Path traversal when writing artifacts using mission-derived paths. | High | Mitigated | UUID-validated mission id + artifacts-root containment check (SR-13/14) + test. |
| R-006 | Module-boundary erosion in a monolith over time. | Low | Open | MODULE_BOUNDARIES.md dependency rule; code review; import-linter planned (Phase 2). |
| R-007 | Floating-point nondeterminism across platforms breaks "reproducible" claim. | Low | Mitigated | Compare with tolerances; record sgp4/numpy versions in sidecars; deterministic algorithm. |
| R-008 | Dependency supply-chain / unpinned upgrades. | Med | Open | Version constraints in pyproject; `--only-binary` installs; dependency review + secret scanning planned in CI. |
| R-009 | Quantum adapter accidentally invoked on the mission path (perf/appearance). | Low | Mitigated | Slice does not import `quantum`; self-test is manual; capabilities only reports availability. |
| R-010 | mypy/ruff/pytest tool versions on 3.14 behave unexpectedly. | Low | Mitigated | Tools installed and run; results reported with exact output at phase boundaries. |
| R-011 | Secrets accidentally committed. | High | Mitigated | `.gitignore` excludes `.env`, DBs, artifacts; only `.env.example` tracked; no secrets in code. |
| R-012b | CelesTrak **legal / commercial-use / redistribution rights** are not established by the GP-data documentation reviewed. | Med | **Open** | Policy keeps `requires_review=true`, `commercial_use_confirmed=false`; attribution recorded; no commercial rights claimed. Requires owner review of CelesTrak's terms before redistribution/commercial use. (Supersedes R-012.) |
| R-013 | External (CelesTrak) data could be misread as live/authoritative. | High | Mitigated | Freshness states (test-fixture/current/fresh/aging/stale/expired/unavailable/invalid); provenance + cache status on every external-source result; no silent fallback to sample data (ADR-0010). |
| R-014 | Uncontrolled refresh storm / over-polling the external source. | Med | Mitigated | Minimum refresh interval enforced per source policy; in-process lock; cache-first reads; no background polling daemon (ADR-0010). |
| R-015 | **Data-rights heterogeneity** across future sources (NASA Earthdata, arXiv, Crossref, Bhuvan, MOSDAC, SatNOGS) — differing access/branding/caching/reuse terms (per both references). | Med | Open | The typed `SourcePolicy`+`SourceLicenseRecord` registry (Phase 2) defaults each source to `requires_review=true`; every new connector must record explicit rights before live use (`DATA_RIGHTS_AND_SOURCE_POLICY.md`). |
| R-016 | Expanded / **Alpha-5 (alphanumeric) catalogue numbers** not supported by request validation (numeric `^\d{1,9}$` only). | Low | Open | Documented in `CELESTRAK_VERIFICATION.md`; numeric ids (incl. expanded range) work; Alpha-5 is a future non-blocking enhancement (`sgp4.alpha5` exists). |
| R-017 | **Scope-discipline / over-reach** — references warn that building the full "universal" vision at once becomes fragile. | Med | Mitigated | Phased roadmap with a narrow first mission; "knows everything / living / self-evolving" translated to bounded, reviewer-gated capabilities (`REFERENCE_RECONCILIATION.md`). |
| R-018 | `qiskit-optimization` community package is no longer officially IBM-supported (per the main reference). | Low | Open (future) | Keep the optimization model representation inside OrbitMind; compile to the Qiskit execution layer only at Phase 4; do not make `qiskit-optimization` a hard dependency (noted for ADR-0005 review). |
| R-019 | **JPL/CNEOS rate limits and reuse/commercial rights** are not confirmed by the docs reviewed (Phase 3A). | Med | Open | Network off by default; conservative cache (24h) + 1h min refresh; `requires_review=true`, `commercial_use_confirmed=false`; attribution recorded (ADR-0014/0015, `DATA_RIGHTS_AND_SOURCE_POLICY.md`). Review official JPL terms before redistribution/commercial use. |
| R-020 | Small-body **orbit solutions carry uncertainty** (short observation arc / high condition code); could mislead. | Med | Mitigated | Deterministic verification (eccentricity/period/angular/arc checks), condition-code preserved, epistemic labels + explicit `limitations`; validation proves only internal consistency, not orbit correctness. |
| R-021 | **Close approach misread as impact prediction.** | High | Mitigated | Every CAD result states nominal distance is not a guarantee and a close approach is NOT an impact; hazard flags are source-reported (never computed); no independent collision probability computed (ADR-0015). |
| R-022 | Alpha-5 / expanded **small-body designations** not accepted (approved formats only). | Low | Open | Documented (`SMALL_BODY_INTELLIGENCE.md`); numeric/standard identifiers work; alphanumeric expansion is a future non-blocking enhancement. |
| R-023 | Small-body **visualization mistaken for a precision prediction**. | Med | Mitigated | Artifacts labelled `model-estimate` with explicit "NOT Horizons / no perturbations / not a prediction" disclaimers + documented coordinate assumptions (ADR-0017). |
| R-024 | Growing **SQLite→PostgreSQL migration surface** (Phase 3A adds 8 tables) before Phase 3B. | Low | **Mitigated (Phase 3B)** | Additive, non-destructive tables behind repository interfaces (ADR-0003); PostgreSQL is now the production system-of-record with dialect-aware code; the port remains a `ORBITMIND_DATABASE_URL` change (ADR-0018). |
| R-025 | **Retrieval dialect drift** — SQLite (exact-term) and PostgreSQL (stemmed full-text) select different candidate sets, so SQLite test results may not match production ranking. | Med | Mitigated | One identical ranking formula on both dialects; only candidate *selection* differs and is labelled (`RetrievalBackend`) on every result; the PostgreSQL FTS path is asserted with a stub test; documented (ADR-0021, DETERMINISTIC_RETRIEVAL.md). PostgreSQL integration tests skip cleanly when no service is present. |
| R-026 | **Memory misread as verified truth** — a stored source passage or registered claim treated as a verified fact. | High | Mitigated | Claims default to `source-asserted`/`assumption`, never `verified-fact`; results carry `not-verified` + explicit limitations; every memory API response carries a "memory is not truth; retrieval is not verification" disclaimer (ADR-0019/0020). |
| R-027 | **Accidental secret / unsafe ingestion** — secrets or arbitrary files entering memory. | High | Mitigated | Allowlisted roots, secret-name + extension + size rejection, path-traversal containment, strict-UTF-8, no execution of contents; proven by `test_memory_ingestion.py` security tests (ADR-0019, INGESTING_DOCUMENTS.md). |
| R-028 | **Citation/version confusion** — a citation silently pointing to a newer document version. | Med | Mitigated | Citations are version-pinned (`version_id` + `chunk_id` + checksum + char range); changed content creates a new immutable version; verified by retrieval tests (ADR-0020). |
| R-029 | **Premature/fragile vector retrieval** — embeddings/pgvector adding network, model downloads, or non-determinism to the default path. | Low | Mitigated | Embeddings/vector search are an optional, **null + disabled** boundary by default; no network/model/key/GPU/pgvector required for standard tests; exact lexical/full-text retrieval is the supported fallback (ADR-0022, VECTOR_RETRIEVAL_BOUNDARY.md). |
